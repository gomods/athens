>注意：本文档介绍的过滤文件已弃用。请参阅["使用下载模式文件进行过滤"](/configuration/download)获取有关如何在 Athens 中过滤模块的最新说明。

代理支持以下三种用例：

1. 直接从源（上游代理）获取模块
2. 排除特定模块
3. 在本地代理中包含模块

这些设置通过创建配置文件完成。在 `config.dev.toml` 中设置 `FilterFile`，或设置 `ATHENS_FILTER_FILE` 环境变量指向该配置文件即可。

### 编写配置文件

配置文件的每一行可以以下列内容开头：

* `+` 表示该模块必须被代理包含
* `D` 表示该模块必须直接从上游代理获取，不在本地存储
* `-` 表示该模块被排除，不会从上游代理获取

支持使用 `#` 添加注释，换行符会被跳过。其他任何内容都会导致错误。

### 示例配置文件

<pre>
# 这是注释


- github.com/azure
+ github.com/azure/azure-sdk-for-go

# 直接获取 golang tools
D golang.org/x/tools
</pre>

上例中，`golang.org/x/tools` 直接从上游代理获取。除了 `github.com/azure/azure-sdk-for-go` 外，`github.com/azure` 的所有模块都被排除。

### 添加默认模式

模块列表可能很快变得很大，有时可能只想为少数模块指定配置。这种情况下，可以为所有模块设置默认模式，再为特定模块添加要应用的规则。默认规则在文件开头指定，可以是 `+`、`-` 或 `D`。

默认模式示例如下：

<pre>
D
- github.com/manugupt1/athens
+ github.com/gomods/athens
</pre>

上例中，所有模块都直接从源获取。`github.com/manugupt1/athens` 被排除，`github.com/gomods/athens` 存储在代理中。

### 向过滤器添加版本

使用"批准列表"是常见做法，需要在每个次要版本或补丁版本被允许进入代码库之前批准它们。这通过向规则添加版本模式列表实现。这些版本模式以逗号分隔且为前缀匹配，因此 `v2` 和 `v2.3.*` 都能匹配请求的版本 `2.3.5`。

版本过滤器示例如下：

<pre>
-
# 直接使用内部 github enterprise server
D enterprise.github.com/company

# 外部依赖批准列表
+ github.com/gomods/athens v0.1,v0.2,v0.4.1
</pre>

上例中，任何不在规则中的模块都会被排除。所有来自 `enterprise.github.com/company` 的模块都直接从源获取。`github.com/gomods/athens` 模块将存储在代理中，但仅限于版本 `v0.4.1` 以及 `v0.1` 和 `v0.2` 次要版本下的补丁版本。

### 版本过滤器修饰符

Athens 提供高级过滤器修饰符，用于处理 API 兼容性或给定依赖项在特定版本更改许可证等情况。修饰符用于过滤文件的模式列表中。

<pre>
-
# 外部依赖批准列表
+ github.com/gomods/athens <v1.2.3
</pre>

目前支持的修饰符有：

* `~1.2.3` 启用从 1.2.3 开始的所有补丁版本（如 1.2.3、1.2.4、1.2.5）
  * 形式上为 `1.2.x`，其中 `x >= 3`

* `^1.2.3` 启用从 1.2.3 开始的所有补丁和次要版本（如 1.2.4、1.3.0 和 1.4.5）
  * 形式上为 `1.x.y`，其中 `x >= 2` 且 `y >= 3`

* `<1.2.3` 启用低于 1.2.3 的所有版本（如 1.2.2、1.0.0 和 0.58.9）
  * 形式上为 `x.y.z`，其中 `x <= 1`、`y <= 2` 且 `z < 3`

这类修饰符仅在指定三部分语义版本时才能工作。例如，`~4.5.6` 可以工作，而 `~4.5` 则不行。
