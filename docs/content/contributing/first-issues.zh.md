---

以下内容旨在帮助项目维护者在处理问题时将问题标记为"适合新手"。这只是指南，因为每个问题都有其特殊性。最终由维护者自行决定。

本文档假设这可能是贡献者的首个 Pull Request 到 Athens 项目。

> 注意：虽然这是为维护者编写的，但任何以帮助新手贡献者为目标编写问题的人都欢迎使用此模板作为指南。

---

通常，我们仍然希望遵循[错误](https://github.com/gomods/athens/blob/main/.github/ISSUE_TEMPLATE/bug_report.md)或[功能](https://github.com/gomods/athens/blob/main/.github/ISSUE_TEMPLATE/feature_request.md)的问题模板。如果您正在进行问题分类，可能需要添加更多信息以满足以下模板。

## 模板

### 初次接触 Athens？

如果您是项目新手，请先花些时间阅读我们的[贡献文档](https://docs.gomods.io/contributing)。

### 需要做什么？

问题应包含错误或功能的详细概述。

应包含项目和外部任何相关参考资料的链接。其他问题、Pull Request 或评论的链接也可以。还可能包括我们文档的相关链接或博客文章。

尝试提供好的起点想法，或任何已经尝试过的内容。

#### 开源新手

如果您是开源、Git 或 GitHub 的新手，或者只是想要一些工作流程提示，请参阅我们的[新手贡献者指南](https://docs.gomods.io/contributing/new)。
